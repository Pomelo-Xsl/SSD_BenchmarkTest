# NVMe SSD Benchmark MVP 操作说明

本文说明如何在 Linux 测试机上启动并验证 NVMe SSD Benchmark MVP。请仅使用专用测试机和测试盘。

> **重要警告：** 写入类 fio 测试会覆盖目标 NVMe SSD 上的数据。不要对系统盘、已挂载磁盘或任何含有重要数据的磁盘执行测试。

## 1. 前置条件

- Linux 系统（建议 Ubuntu/Debian）。
- Python 3.11 或更新版本。
- 用于测试的独立 NVMe SSD，且设备及其分区均未挂载。
- 启动实际测试服务时具有 `root` 权限。

## 2. 安装依赖

进入项目目录并创建 Python 虚拟环境：

```bash
cd "/Users/pomelo/Documents/New project"

python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

安装系统工具（Ubuntu/Debian 示例）：

```bash
sudo apt update
sudo apt install -y fio nvme-cli util-linux
```

确认工具可用：

```bash
fio --version
nvme version
lsblk --version
```

## 3. 启动服务

实际执行 fio 前，服务必须以 root 身份运行；否则任务会被安全检查拒绝。

```bash
cd "/Users/pomelo/Documents/New project"
sudo "$(pwd)/.venv/bin/uvicorn" app.main:app --host 127.0.0.1 --port 8000
```

服务启动后可访问交互式 API 文档：

```text
http://127.0.0.1:8000/docs
```

## 4. 验证服务健康状态

另开一个终端，执行：

```bash
curl http://127.0.0.1:8000/health
```

预期响应：

```json
{"status":"ok"}
```

## 5. 扫描 NVMe 设备

```bash
curl http://127.0.0.1:8000/api/devices
```

响应中会列出 NVMe 设备名称、路径、型号、序列号、容量、固件和温度等信息。例如：

```json
[
  {
    "name": "nvme1n1",
    "path": "/dev/nvme1n1",
    "model": "Example SSD",
    "serial": "ABC123",
    "size_bytes": 1000204886016,
    "firmware": "1.0",
    "temperature_c": 35.0
  }
]
```

在创建任务前，务必人工确认目标不是系统盘，并检查它没有挂载分区：

```bash
lsblk -o NAME,PATH,MOUNTPOINTS
findmnt /
```

## 6. 运行只读测试（推荐先执行）

将下面的 `nvme1n1` 替换为扫描结果中的设备名。以下测试不会写入磁盘：

```bash
curl -X POST http://127.0.0.1:8000/api/tasks \
  -H 'content-type: application/json' \
  -d '{"device_name":"nvme1n1","test_name":"rand_read_4k"}'
```

可选的只读测试类型：

- `seq_read_128k`：128 KiB 顺序读取。
- `rand_read_4k`：4 KiB 随机读取。

成功创建后，接口会返回任务。例如：

```json
{"id":1,"device_name":"nvme1n1","test_name":"rand_read_4k","status":"queued"}
```

## 7. 查询任务和结果

查询单个任务状态：

```bash
curl http://127.0.0.1:8000/api/tasks/1
```

任务状态依次为 `queued`、`running`、`completed`；失败时为 `failed`，并通过 `error_message` 返回原因。

任务完成后查询指标：

```bash
curl http://127.0.0.1:8000/api/results/1
```

结果包括：

- `iops`：每秒 I/O 次数。
- `bw_mib_s`：带宽（MiB/s）。
- `latency_avg_us`：平均延迟（微秒）。
- `latency_p99_us`：P99 延迟（微秒）。
- `cpu_user_pct`、`cpu_system_pct`：fio 进程 CPU 使用率。

所有任务也可通过以下接口查看：

```bash
curl http://127.0.0.1:8000/api/tasks
```

## 8. 写入测试（危险）

仅可在已确认为空的专用测试盘上执行。写入任务必须显式设置 `confirm_destructive` 为 `true`：

```bash
curl -X POST http://127.0.0.1:8000/api/tasks \
  -H 'content-type: application/json' \
  -d '{"device_name":"nvme1n1","test_name":"rand_write_4k","confirm_destructive":true}'
```

可选的写入测试类型：

- `seq_write_128k`：128 KiB 顺序写入。
- `rand_write_4k`：4 KiB 随机写入。

服务会在运行 fio 前检查 root 权限、设备存在性、NVMe 设备类型、挂载状态、系统盘风险以及 fio/nvme-cli 是否已安装。任一检查失败，任务将标记为 `failed`。

## 9. 查看文件与日志

- SQLite 数据库：`benchmark.db`。
- fio 原始 JSON：`results/task_<任务ID>.json`。
- 应用日志：`logs/app.log`。
- fio 运行日志：`logs/fio.log`。
- 错误日志：`logs/error.log`。

遇到问题时，优先查看任务接口的 `error_message` 和 `logs/error.log`。
